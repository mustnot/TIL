## 데이터 시스템의 옵션들

최근 마케팅, CRM, ERP 등 다양한 플랫폼 및 소프트웨어들은 API를 통해 데이터를 주고 받을 수 있는 환경을 구축하여 하나의 생태계를 생성하고 있다. 이렇게 구축된 생태계는 유저들로 하여금 직접 데이터를 받아 이용할 수 있게하거나 입력할 수 있게하여 보다 풍부한 데이터 생태계를 구성할 수 있다.

<br>

### Relational Databases

**Relational Databases**란 관계형 데이터베이스를 말하는데 이는 데이터의 **관계도**를 기반으로 한 디지털 데이터베이스를 의미하며 데이터의 저장을 목적으로 생겨났다. SQL이라고 하는 스탠다드 방식을 통해 자료(데이터)를 열람하고 유지할 수 있게 개발되었다. 관계형 데이터베이스가 지금 중요한 이유는 현재 대부분의 서비스들이 가장 많이 쓰고 있는 데이터 시스템이기 때문이다. RDB라고 말이 말하는데, 대표적으로 MySQL, PostgreSQL, SQL Server, SQLite 등이 이에 속한다.



### NoSQL Databases

NoSQL (non SQL) 데이터베이스는 관계형 데이터베이스보다 덜 제한적인 일관성 모델을 이용하는 데이터 저장 및 검색을 위한 매커니즘을 제공한다. 이렇게 개발된 동기에는 디자인의 단순화, 수평적 확장성, 세세한 통제를 포함한다. NoSQL 데이터베이스는 단순 검색 및 추가 작업을 위한 매우 최적화된 키 값 저장 공간으로 레이턴시와 스루풋과 관련하여 상당한 성능 이익을 내는 것이 목적이다. 주로 빅데이터와 실시간 웹 애플리케이션의 상업적 이용에 널리 쓰이는데, 그러다보니 메신저 앱에도 사용된다. 또한 NoSQL 시스템은 SQL 계열 쿼리 언어를 사용할 수 있다는 사실을 강조하는데 그러다보니, "Not Only SQL"이라 불리기도 한다.



### Hadoop / Spark / Presto 등 빅데이터 처리

Hadoop과 Spark, Presto는 Distributed Storage System / MapReduce를 통한 병렬 처리를 이용하는데, 빅데이터 시스템에 많이 쓰이는만큼 병렬처리에 집중된 이유는 데이터가 커지게 되면서 지금까지 사용되어왔던 직렬적인 (일차원적인) 처리 방식은 속도 측면에서 매우 비효율적이기 때문에 데이터를 병렬로 나누어 각각의 노드마다(?) 배치(?), 작업을 분배하여 보다 효율적인 처리를 하도록 되어 있다.

그 중 Spark는 보통 빅데이터 분석 환경에 최적화되어 있어 Real Time 데이터를 프로세싱하는 경우 자주 이용되는데, 최근 많이 사용되는 이유는 속도가 큰 영향을 주었지만, 여러 API를 제공한다는 점이다. Java 뿐만 아니라 분석에 자주 사용되는 Python, Scala를 통해 애플리케이션 생성을 할 수 있고, SQL Query 환경을 제공하여 분석가들에게 각광 받고 있다.



### Serverless Framework

서버리스 프레임워크는 지난번에 한번 공부했지만 최근 많이 이슈가 되는 이유는 관리에 용이성이다. 실제로 서버를 구매하여 서버실에 구비해놓는건 현실적으로 많은 비용과 시간이 소요되는 반면에 서버리스 프레임워크는 이런 비용과 시간을 단축시켜주고 서버를 세팅하는 과정이나 계정과 권한을 생성해서 전달하는 건 비효율적이기 때문에 이를 해결하기 위해 클라우드에서 제공하는 서버리스 프레임워크를 사용한다.

장점은 Pay as you use로 사용한만큼 지출되는 비용 그리고 Form of functions 함수 형태의 폼도 제공한다는 점 (Lambda와 같은) 또한 써드파티 앱들과 다양한 API를 쉽게 적용할 수 있기 때문에 데이터를 수집하거나 정제하는데 매우 유용하기에 데이터 엔지니어링에 큰 부분을 차지하고 있다.

## 데이터 파이프라인

> 데이터 파이프라인이란, 데이터를 한 장소에서 다른 장소로 옮기는 것을 의미한다.

데이터 파이프라인이 필요한 이유는 빅데이터라고 말할 수 있을만큼 현재 다양한 데이터 소스들이 기업에 유입되고 각각 이용되고 있는데, 이러한 다양한 소스들로부터 많은 데이터를 생성하고 저장하기 위해서는 하나의 데이터 파이프라인이 필요하게 되었다. 주로 데이터가 서로 고립되어 있는 경우 사용되는데, 어떠한 서비스나 플랫폼을 운영한다고 한다면, 그로 인해서 생성되는 수 많은 데이터들이 현실적으로는 한 곳에서 생성되고 한 곳으로 저장되진 않는다. 이것을 **데이터 사일로**라고 말하는데, 마케팅, 어카운팅, 세일즈, 오퍼레이션 등 각 영역의 데이터가 서로 고립되어 있는 경우에 데이터 파이프라인이 필요하다고 말한다.@또한 실시간 혹은 그 보다 높은 수준의 데이터 분석이 필요한 비즈니스 모델을 가진 사업이라면 보다 효율적인 데이터 파이프라인 시스템이 필요하게 된다.

데이터 파이프라인을 구축하기 위해 고려해야할 사항이 있다.

1. Scalabaility : 예상보다 데이터가 기하급수적으로 늘어났을 때에도 작동이 원활한가?
2. Stability : 에러, 데이터 플로우 등 다양한 이벤트에 대한 모니터링 관리
3. Security : 데이터 이동간 보안에 대한 리스크는 무엇인가?
