## Gettings Started

Go는 범용 프로그래밍 언어로 깔끔하고 간결하며 생산성 높은 프로그래밍 언어이다. 작성한 코드를 빠르게 컴파일하고 가비지 컬렉션을 지원한다. 정적 자료형 언어지만 동적 자료형 언어처럼 쉽게 개발이 가능하며 동시성을 지원하는 코드 역시 쉽게 작성이 가능하다.

특징을 설명하면,

- 부분적이지만, 편리한 자료형 추론을 제공하여 굳이 반복해서 자료형 이름을 쓰지 않아도 된다.
- 소스 코드 형식을 자동으로 맞춰주는 도구 및 여러 편리한 도구가 기본으로 제공된다.
- Example 테스트를 이용하여 쉽게 테스트 코드를 작성하면서 코드 문서화까지 할 수 있다.
- 함수 리터럴 및 클로저를 자유자재로 사용할 수 있다.
- 명시적으로 인터페이스를 지정하지 않아도 인터페이스 구현이 가능하여 기존에 있던 코드를 고치지 않고도 유연한 구현이 가능하다.
- 채널을 이용하여 동시성 구현을 락 등을 이용하지 않고 간편하게 할 수 있으며, 언어 고유의 지원으로 교착 상태나 경쟁 상태를 파악하기 쉽다.
- 컴파일 속도가 빨라서 컴파일 및 테스트를 반복적으로 수행하면서 코드를 작성하기 용이하다.
- 가비지 컬렉션 지원으로 메모리 관리에 대한 부담을 덜 수 있다.
- 자료형 리터럴을 쉽게 쓸 수 있다.

<br>

## Installation

1. [Go lang](https://golang.org/) 사이트에서 설치 할 수 있다.
2. 설치가 완료된 후에는 `go version` 을 통해 확인이 필요하다.

```bash
$ go version
go version go1.15.5 darwin/amd64
```

3. `go env` 명령어를 통해 설치된 위치를 확인

```bash
$ go version
..
GOPATH="$HOME/go"
GOROOT="/usr/local/go"
..
```

(더 많은 내용이 포함되어 있지만, 아직 초심자에게는 `GOPATH`, `GOROOT` 만 확인하고 가자)

* `GOPATH` : 작업 디렉토리로 이해하면 좋다. 해당 경로를 통해 다양한 패키지들이 설치되고 저장된다.
* `GOROOT` : `go` 컴파일러의 위치이다.

<br>

## Hello World

1. GO PATH로 이동

```bash
cd $GOPATH
```

(만약 `GOPATH`를 변경하고 싶다면 `export GOPATH={directory}` 명령어로 새롭게 설정하면 된다)

2. 기본 지정된 경로라면, `bin, src, pkg` 폴더가 생성되어 있고 그 안에 패키지와 기타 파일들이 있다.
3. `main.go` 생성

```go
package main

import "fmt"

func main() {
	fmt.Println("Hello, World!")
}
```

4. `go run main.go` 실행

```bash
$ go run main.go
Hello, World!
```

* 실행 순서
  * `package main`
  * `import`
  * `func main()`

## Go 기본 사용법

### 자료형 및 변수 

```go
var x int
```

Go의 변수와 자료형 선언은 `var x int` 순서로 `변수 x는 int형이다`와 같이 말하기 자연스러운 순서로 되어 있다. 또한 Go 역시 파이썬과 비슷하게 자료형과 변수 선언문 역시 제외해도 동작에 아무런 문제가 없다.

```go
var i = 10
var p = &i // i의 주소값을 p에 저장

i : 10
p := &i
```

하지만 Python과 차이점이 있는데 C++과 마찬가지로 Go는 가장 먼저 선언된 자료형에 따라 자료형이 정해지기 때문에 이미 int 형으로 정해진 변수에는 문자열을 넣을 수 없다.

<br>

### 함수와 간단한 제어 구조

```go
package main

import "fmt"

func fac(n int) int {
	if n <= 0 {
		return 1
	}
	return n * fac(n-1)
}

func main() {
	fmt.Println(fac(5))
}
```

함수의 형태는 `func 함수명(파라미터) 반환값` 형식을 띄고 있으며, 함수 안에 파라미터로 함수 자체도 받을 수 있다.

<br>

### 디렉터리 구조

* bin : 바이너리 파일들이 저장되며 실행 파일들이 들어간다.
* pkg : 패키지 오브젝트 파일들이 들어간다. 소스가 컴파일된 후의 코드들이 여기 들어가는 것이지만, 실행 가능한 파일들은 아니다. 라이브러리가 들어가 있다고 보면 된다. (non executable library)
* src : 소스 코드들이 저장되는 디렉터리로 자신만의 패키지 경로를 작성하면 된다. 일종의 repository 같은 역할이다.
