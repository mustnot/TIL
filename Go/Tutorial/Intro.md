# Go 언어란

Go언어는 구글이 개발한 프로그래밍 언어로 2007년 켄 톰슨, 롭 파이크, 로버트 그리즈머가 최초 설계를 시작으로 2008년부터 본격적으로 개발된 언어이다. 특히 C 언어의 모체가 된 B 언어와 유닉스 개발자로 유명한 켄 톰슨이 참여한 언어이다.

Go 언어는 범용 프로그래밍 언어로 빠른 성능, 안정성, 편의정, 쉬운 프로그래밍을 목표로 개발되었다.

특징은 다음과 같다.

* 정적 타입, 강 타입
* 컴파일 언어
* 가비지 컬렉션
* 병행성(Concurrency)
* 멀티코어 환경 지원
* 모듈화 및 패키지 시스템
* 빠른 컴파일 속도

Go 언어는 C 언어를 기반으로 하여, C++의 복잡한 문법 대신 간단하고 간결한 문법을 추구한다.

<br>

### 정적 타입과 동적 타입

프로그래밍 언어에서 타입이란 우리말로 자료형을 뜻하는데, 이러한 자료형에는 일반적으로 정수, 실수, 불, 문자열, 객체 등이 있다. 여기서 자료형은 컴파일할 때 결정하면 <u>정적 타입</u>이고, 실행할 때(Run-time) 결정하면 <u>동적 타입</u>이라 한다.

최근 유행하고 있는 Python, Ruby, JavaScript 등의 스크립트 언어는 동적 타입(Dynamic Typing) 언어이다. 따라서 동적 타입 언어의 변수는 자료형이 정해져 있지 않아 실행할 때 결정된다.

반면에 C, C++, Java 등 언어에서는 정적 타입(Static Typing) 언어로 값의 형태에 따라 각각 자료형을 가지며 자료형에 맞지 않는 값을 대입하면 컴파일 할 때 에러가 발생한다.

<br>

### 약 타입과 강 타입

약 타입(Weakly-typed)은 값의 타입을 바꿀 수 있다. 여기서 값 타입 바꾸기란 형 변환이라고 하는데, 이렇듯 약 타입 언어는 자료형이 달라도 컴파일 또는 실행 시점에 정해진 규칙에 따라 암시적 형 변환(Implicit conversion)을 해주는 방식이다.

반대로 강 타입(Strongly-typed)은 값 자체가 타입이며, 타입을 바꿀 수 없다. 즉 컴파일 또는 실행 시 자료형이 다르면 에러를 발생시킨다. Go 언어는 이 중 강 타입 언어로 암시적 형 변환을 하지 않고 컴파일할 때 타입을 결정하므로 정적 타입의 언어이다.
